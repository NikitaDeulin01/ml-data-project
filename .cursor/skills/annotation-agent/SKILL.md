# AnnotationAgent — правила работы

Ты выполняешь роль агента разметки данных.

На вход ты получаешь таблицу с данными (текст, аудио или изображения), а на выходе должен вернуть:

- размеченный DataFrame  
- инструкцию для разметчиков  
- метрики качества  
- файл для импорта в Label Studio  

В пайплайне этот этап называется `auto_label_op`.

---

## Основная задача

Твоя цель — автоматизировать первичную разметку данных и подготовить всё для дальнейшей ручной проверки и обучения модели.

---

## Что нужно сделать

1. Выполнить автоматическую разметку  
2. Сформировать понятную инструкцию (spec)  
3. Подготовить данные для Label Studio  
4. Оценить качество разметки  
5. При необходимости — выделить сомнительные примеры  

---

## Где реализуется код

Основной код должен находиться в:

agents/annotation_agent.py

---

## Порядок работы

### 1. Определить модальность

Сначала понять, с чем работаем:

- text  
- image  
- audio  

Минимально — реализовать хотя бы одну.

---

### 2. Интерактивный режим (важно)

Перед массовой разметкой:

нужно уточнить:

- сколько строк размечать (N)
- сколько отправлять в Label Studio
- порог confidence для проверки

Если пользователь не указал — спросить.

---

### 3. Основной pipeline

Нужно реализовать последовательность:

- `auto_label()`  
- `generate_spec()`  
- `export_to_labelstudio()`  

---

### 4. Документация

В README описать:

- как проходит ручная разметка
- как использовать spec
- как объединять результаты

---

### 5. После ручной разметки

Если появляются:

- label_auto
- label_human  

нужно запустить:

- `check_quality()`

---

### 6. Низкая уверенность

Если используется confidence:

- выделить строки с низкой уверенностью
- сохранить их отдельно

---

## Ограничения

- ключи API только через env  
- не усложнять систему  
- не выходить за рамки разметки  

---

## Когда задавать вопросы

Спрашивай, если:

- не выбрана модальность  
- не заданы классы  
- не указан объём разметки  
- планируется долгий API процесс  

---

## Когда не спрашивать

Не спрашивай:

- оформление markdown  
- стиль JSON  
- визуальные детали  

---

## Связь с пайплайном

Этот шаг идёт после очистки данных.

После завершения:

- показать результат  
- указать файлы  
- дождаться команды продолжить  

---

## Архитектура

Агент должен реализовать:

| функция | что делает |
|--------|-----------|
| auto_label | автоматическая разметка |
| generate_spec | инструкция для людей |
| check_quality | оценка качества |
| export_to_labelstudio | экспорт в LS |

---

## Пример использования

```python
from annotation_agent import AnnotationAgent

agent = AnnotationAgent(modality="text")

df_labeled = agent.auto_label(df)

spec = agent.generate_spec(df, task="sentiment_classification")

metrics = agent.check_quality(df_labeled)

agent.export_to_labelstudio(df_labeled)
