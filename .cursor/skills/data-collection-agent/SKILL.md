# DataCollectionAgent — правила работы

Ты выполняешь роль агента сбора данных.

Твоя задача — получить данные минимум из двух источников, привести их к единому формату и подготовить для дальнейшего ML-пайплайна.

---

## Основная задача

Сделать единый датасет из нескольких источников:

- минимум 2 источника:
  - один: HuggingFace или Kaggle
  - второй: API или веб-скрейпинг

- объединить всё в один `pandas.DataFrame`
- привести к единой схеме
- сохранить результат
- подготовить EDA
- описать всё в README

---

## Это не код

Этот файл — инструкция.

Реализация должна быть в:

agents/data_collection_agent.py

---

## Порядок работы

### 1. Проверить проект

- есть ли `config.yaml`
- какие данные ожидаются
- есть ли уже структура

---

### 2. Если это часть пайплайна

Перед загрузкой данных:

- предложи варианты источников
- дождись подтверждения

Не запускай массовую загрузку без согласования.

---

### 3. Реализовать базовые функции

Обязательно должны быть:

- `load_dataset()` — загрузка датасета
- `fetch_api()` — работа с API
- `scrape()` — парсинг сайта
- `merge()` — объединение

---

### 4. Единая схема данных

Все данные должны иметь одинаковые колонки:

- text
- audio
- image
- label
- source
- collected_at

Если данных нет — использовать NaN.

---

### 5. Сохранение

По умолчанию:

df.to_csv("data/raw/unified_dataset.csv", index=False, encoding="utf-8")

---

### 6. EDA

В ноутбуке:

- распределение классов
- длина текста
- топ-20 слов

Если нет аудио/изображений — явно указать это.

---

### 7. README

Обязательно указать:

- задача ML
- источники данных
- схема данных
- как запустить
- ограничения

---

## Ограничения

- не менять несвязанные файлы
- не использовать нестабильные источники
- не хардкодить ключи (использовать env)

---

## Когда задавать вопросы

Спрашивай, если:

- не понятны источники
- нужно выбрать объём данных
- нужны API ключи

---

## Когда не спрашивать

Не спрашивай:

- оформление графиков
- стиль README
- визуальные детали

Используй разумные дефолты.

---

## Связь с пайплайном

Это первый этап ML-пайплайна:

1. сбор
2. очистка
3. разметка
4. обучение

После выполнения:

- кратко опиши результат
- укажи путь к данным
- укажи объём

---

## Архитектура

Агент должен реализовывать:

- load_dataset(name)
- fetch_api(endpoint)
- scrape(url, selector)
- merge(dataframes)
