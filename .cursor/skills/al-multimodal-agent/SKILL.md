# Active Learning и работа с мультимодальными данными

После этапа разметки в ML-пайплайне обычно идёт шаг активного обучения.

Здесь используется агент `ActiveLearningAgent`, который:

- выбирает наиболее информативные примеры из пула
- сравнивает стратегию с случайным выбором (random)
- строит кривые обучения
- оценивает эффективность

Если задача требует объединения разных типов данных (например текст + изображения), используется отдельный агент — `MultimodalAgent`.

Важно: не объединяй AL и мультимодальность в одном классе без явного запроса.

---

## Общая логика

Агент может работать в двух режимах:

1. только Active Learning  
2. Active Learning + объединение модальностей  

Перед началом нужно определить, какой режим нужен.

---

## Порядок работы

### 1. Уточнение задачи

Перед запуском:

- нужен ли только AL  
- или ещё объединение модальностей  

---

### 2. Интерактивный режим

Если не включён autopilot:

нужно уточнить:

- стратегию отбора (`entropy`, `margin`, `random`)
- количество итераций (`n_iterations`)
- размер батча (`batch_size`)
- размер начальной выборки  

---

### 3. Active Learning

Основной процесс:

- запуск `run_cycle` с выбранной стратегией  
- отдельный запуск с `random`  
- построение одного графика  

оси:

- X — количество размеченных объектов  
- Y — метрика (accuracy / F1)  

после этого:

- кратко объяснить результат  
- показать, есть ли выигрыш относительно random  

---

### 4. Мультимодальность (если требуется)

Если пользователь просит:

- загрузить каждую модальность  
- объединить их по ключу  
- явно обработать несовпадения  
- провести базовый анализ  
- сохранить результат  

---

## Ограничения

- для entropy требуется `predict_proba`  
- если его нет — нужно fallback решение  
- нельзя терять строки при merge без объяснения  

---

## Когда задавать вопросы

Спрашивай, если:

- не задана стратегия AL  
- не заданы параметры цикла  
- не понятно, нужна ли мультимодальность  
- не указан тип метрики (macro / binary F1)  

---

## Когда не спрашивать

Не спрашивай:

- стиль графиков  
- оформление  
- визуальные детали  

---

## Связь с пайплайном

Этот этап идёт после:

- разметки  
- проверки человеком  

Мультимодальность выполняется только если явно запрошена.

---

# ActiveLearningAgent — контракт

Агент должен реализовать:

| метод | что делает |
|------|-----------|
| fit | обучение модели |
| query | выбор новых примеров |
| evaluate | оценка качества |
| run_cycle | полный цикл AL |
| report | построение графика |

---

## Стратегии выбора

Обязательно поддерживать:

- entropy  
- margin  
- random (baseline)  

---
